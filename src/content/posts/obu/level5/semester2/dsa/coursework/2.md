## Introduction 

This coursework covers binary search trees, stacks, queues, graphs, and path-finding algorithms. The objective is to develop a comprehensive understanding and practical implementation of these data structures and algorithms and to apply them to solve relevant problems.

### *Task 1: Binary Search Tree Implementation*

Design and implement a binary search tree data structure.

- The binary search tree should be a generic data structure that can store elements of integer data type.
- Implement a `TreeNode` class that represents each node in the binary search tree. Each node should contain the data, a reference to the left child node, and a reference to the right child node.
    1. The node class should have appropriate constructors and accessor methods.
    2. It should maintain the integrity of the binary search tree structure and the ordering property.
- Implement a `BinarySearchTree` class that represents the binary search tree itself. It should include methods for inserting nodes, deleting nodes, performing pre-order traversal, in-order traversal, post-order traversal, searching for a specific value, and returning the height of the tree.
    1. For the pre-order traversal method, it should visit the root node first, then recursively traverse the left subtree, and finally the right subtree. The method should return a list of the visited nodes in the preorder sequence.
    2. For the in-order traversal method, it should recursively traverse the left subtree first, then visit the root node, and finally the right subtree. The method should return a list of the visited nodes in the in-order sequence.
    3. For the post-order traversal method, it should recursively traverse the left subtree first, then the right subtree, and finally visit the root node. The method should return a list of the visited nodes in the post-order sequence.
    4. The insertion operation should maintain the binary search tree property (i.e., for each node, all values in the left subtree are less than the node's value, and all values in the right subtree are greater than the node's value). In case of duplicate values, handle them according to a predefined rule (e.g., allow duplicates and insert them either to the left or right subtree based on a specific criterion).
    5. The deletion operation should handle different cases (leaf node, node with one child, node with two children) properly while maintaining the binary search tree property.
    6. The search operation should return the node if the value is found or null otherwise.
    7. The height calculation method should return the correct height of the tree.
- The implementation should include error handling to handle errors such as deleting a non-existent node and searching for a value in an empty tree.
    1. Provide meaningful error messages to help users understand the problem.
- Test the implementation.
    1. Create a set of 500 random integer data in an unordered sequence. 
    2. Create an object of the `BinarySearchTree`.
    3. Using the integer set to test the functionality and correctness of the object.

### *Task 2: Stack and Priority Queue Implementation*

Design and implement a stack and a priority queue data structure.

- The stack should be a Last-In-First-Out (LIFO) data structure that can store elements of the string data type.
    1. Implement a `StackNode` class and a Stack class. The Stack class should include methods for pushing elements onto the stack, popping elements from the stack, checking if the stack is empty, and getting the size of the stack.
    2. The implementation should handle stack overflow and underflow conditions gracefully.
- The priority queue should be a data structure that can store elements of the string data type and prioritize them based on a priority function.
    1. Implement a `PriorityQueueNode` class and a `PriorityQueue` class. The `PriorityQueue` class should include methods for enqueuing elements into the queue, dequeuing elements from the queue, checking if the queue is empty, and retrieving the size of the queue.
    2. Implement a priority function that changes the priority of an element based on an integer value.
    3. The implementation should ensure that elements are dequeued in the correct priority order.
- Test the implementation.
    1. Create an object of the Stack class and an object of the `PriotityQueue` class.
    2. Design test cases to test the functionality and correctness of the object.

### *Task 3: Graph Implementation*

Design and implement a graph data structure.

- The graph should be represented using an adjacency list.
    1. The adjacency list should be implemented by using the doubly linked list structure.
    2. It should handle the storage of vertex and edge information appropriately.
- Design and implement a Graph class that represents the graph. It should include methods for adding vertices, adding edges between vertices, removing vertices, removing edges, checking if two vertices are adjacent, performing depth-first traversal (DFT) and breadth-first traversal (BFT), and using the stack and priority queue implemented in Task 2 for specific graph operations (e.g., using the stack for topological sorting or the priority queue for Dijkstra's algorithm implementation).
    1. The adding vertices method should update the graph structure correctly.
    2. The adding edges method should handle weighted and unweighted edges properly.
    3. The removing vertices method should remove all associated edges as well.
    4. The removing edges method should update the graph structure accordingly.
    5. The adjacent check method should return true or false accurately.
    6. For the DFT method, it should start from a given vertex and explore as deep as possible along each branch before backtracking. The method should return a list of vertices visited in the order of traversal.
    7. For the BFT method, it should start from a given vertex and visit all the vertices at the current level before moving to the next level. The method should return a list of vertices visited in the order of traversal.
- The implementation should handle the case of weighted and unweighted edges.
    1. Store and access the weight information correctly if applicable.
    2. Differentiate between weighted and unweighted operations when necessary.
- The implementation should include error handling to handle errors such as adding an edge between non-existent vertices and removing a non-existent vertex.
    1. Provide appropriate error handling and feedback.
    2. Ensure the graph remains in a consistent state after error handling.
- Test the implementation.
    1. Design a graph that contains at least 10 vertices and more than 30 edges. Generate an object of the Graph class according to the graph you designed.
    2. Design and use test cases to test the functionality and correctness of the object.

### *Task 4: Path Finding Algorithm Implementation*

Implement Dijkstra's algorithm to find the shortest path between two vertices in the graph implemented in Task 3.

- Implement a method within the Graph class to implement these algorithms.
    1. The start and the end vertex for path-finding could be set manually.
- The implementation should use the data structures created in Task 2 if needed.
- The implementation should return the shortest path and its length.
    2. The path should be represented in a suitable format (e.g., a list of vertices).
    3. The length should be calculated accurately.
- The implementation should handle the case as no path between the two vertices.
    1. Return an appropriate indication (e.g., null or a special value) when no path exists.
- The implementation should include error handling to handle errors such as invalid vertex inputs.
    1. Validate the vertex inputs and provide error messages if they are incorrect.
- Test the implementation.
    1. Design and use test cases to test the functionality and correctness of the methods.

### *Task 5 Complexity Comparison and Analysis (Annotation)*

Analyse and compare the time and space complexities of all the implementations in Tasks 1 through 4. Write the analysis and conclusion as a block annotation in each task implementation.

- For the binary search tree, analyze the complexity of insertion, deletion, traversal (pre-order, in-order, post-order), and search operations in different scenarios (e.g., best case, average case, worst case).
- For the graph, compare the complexity of adding vertices, adding edges, removing vertices, removing edges, DFT, BFT, and the path-finding algorithms when applied to different graph types (e.g., sparse graphs, dense graphs).
- Explain the factors that influence the complexity in each case and discuss any trade-offs between different data structures and algorithms.
